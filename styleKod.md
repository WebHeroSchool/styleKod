# My JavaScript Style Guide
## 1. Именование
---
+ Все имена которые мы задаем должны быть в `lowerCamelCase`. Названия переменных, объектов и функций
+ Не нужно использовать однобуквенные имена
+ Названия классов и конструкторов именуй с большой буквы

__Bad__
``` js
const firstuserinfo = {}
function f() {}
const value = new user() {}
```
__Good__
``` js
const firstUserInfo = {}
function getInfo() {}
const value = new User() {}
```

## 2. Переменные
---
+ Объявляйте все локальные переменные либо с помощью `const`, либо с помощью `let`. Используйте `const` по умолчанию, если только переменная не нуждается в переназначении. Ключевое слово `var` не должно использоваться.
+ Каждая декларация локальной переменной объявляет только одну переменную
+ Не объявляйте неиспользуемые переменные.

__Bad__
``` js
var firstUser = 'Admin';
var secondUser = 'Guest';
let a=1, b=2;
let c=3;
function () {
  console.log(`${a} - ${firstUser}`);
  console.log(`${b} - ${secondUser}`);
}
```
__Good__
``` js
const firstUser = 'Admin';
const secondUser = 'Guest';
let a=1;
let b=2;
function () {
  console.log(`${a} - ${firstUser}`);
  console.log(`${b} - ${secondUser}`);
}
```
## 3. Пробелы, отступы
---
+ Используйте отступ в 2 пробела
+ Всегда ставьте пробелы вокруг операторов, ключевых слов, а так же после запятых
+  Не ставьте пробел после открывающей круглой скобки и перед закрывающей круглой скобкой  в теле функций, условий если они написаны в несколько строк. Если написаны в 1 строку, то пробелы нужны.

__Bad__
``` js
function getResult() {.
if(a==b) {result=a+b;};
.}
```
__Good__
``` js
function getResult() {
  if (a == b) { result = a + b; };
}
```

## 4. Запятые, точка с запятой
---
+ Не используйте запятые в начале строки.
+ Всегда ставьте точку с запятой в конце выражения.

__Bad__
``` js
const books = [
  ,book1
  ,book2
  , book3
]
```
__Good__
``` js
const books = [
  book1,
  book2,
  book3,
];
```

## 5. Объекты
---
+ Поместите открывающую скобку на ту же строку, что и имя объекта.
+ Используйте двоеточие плюс один пробел между каждым свойством и его значением.
+ Используйте кавычки вокруг строковых значений,а не вокруг числовых.
+ Добавляйте запятую после последней пары свойство-значение.
+ Поместите закрывающую скобку на новую линию, без начальный пробел.
+ Всегда заканчивайте определение объекта точкой с запятой.

__Bad__
``` js
var person = 
{
  firstName:"Андрей",
  lastName:"Щипунов",
  year:"50",
  eyeColor: "зеленый" }
```
__Good__
``` js
var person = {
  firstName: "Андрей",
  lastName: "Щипунов",
  year: 50,
  eyeColor: "зеленый",
};
```

## 6. Массивы
---
+ Используйте квадратные скобки [ ] для объявления массивов.
+ Не оставляйте пусты места (“дыры”) в массивах.
+ Не ставьте пробел после открывающей квадратной скобки и перед закрывающей квадратной скобкой.

__Bad__
``` js
const arr = new Array();
const books = [ 'book1',, 'book2', 'book3'  ];
```
__Good__
``` js
const arr = [];
const books = ['book1', 'book2', 'book3'];
```


## 7. Функции
### Используйте описательные имена, которые говорят сами за себя.
---
Имя функции должно быть глаголом или фразой, полностью раскрывающими намерение, стоящее за ними, а также намерение аргументов. Их имя должно описывать, что они делают.

__Bad__
``` js
function sEmail(arg) {
  //...
}
```
__Good__
``` js
function sendEmail(emailAddress) {
  //...
}
```
## 8. Функции
### Минимум аргументов функции
---
Cледует избегать большого количества аргументов. Один или два аргумента — идеальный случай, и, если возможно, следует избегать создания трех.  Но если нужно более двух аргументов, то как правило в качестве аргумента будет достаточно объекта, более высокого уровня.

__Bad__
``` js
function createMenu(title, body, buttonText, cancellable) {
  // ...
}
 
createMenu("Foo", "Bar", "Baz", true);
```
__Good__
``` js
function createMenu({ title, body, buttonText, cancellable }) {
  // ...
}
 
createMenu({
  title: "Foo",
  body: "Bar",
  buttonText: "Baz",
  cancellable: true
});
```

## 9. Стрелочные функции
---
+ Используйте стрелочные функции для передачи коллбеков.
+ Если тело функции состоит из одной операции, то можно опустить фигурные скобки и использовать неявный `return`.
+ При использовании стрелочной функции не оборачивай аргумент в скобки, если он один.

__Bad__
``` js
const getNewArr = arr.map(function(item) {
  return item.length;
});

```
__Good__
``` js
const getNewArr = arr.map(item => item.length);
```


## 10. Операторы сравнения
---
+ Используйте === и !== (строгое сравнение ), вместо == и != (нестрогое сравнение).

Если использовать неявное сравнение, то могут возникнуть проблемы из-за 
 неявного преобразования типов данных, поэтому использование строгого сравнения является хорошей практикой. 

__Bad__
``` js
a == b
result == 0
result == true
result == undefined
```
__Good__
``` js
a === b
result === 0
result === true
result === undefined
```
Исключения:
+ сравнение двух литеральных значений:
+ вызов typeof;
+ сравнение с nullю
